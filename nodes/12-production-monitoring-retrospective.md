# 12. 运行监控与复盘

[返回工作流框架](../../AI自动化研发工作流实践指南.md)

## 产物文件

读取发布记录和监控目标，写入 `/Users/j/codes/ai-workflow/tasks/<task-id>/12-monitoring-retrospective.md`，并更新 `task.yaml`。

## 目标

确认发布后的真实运行质量，将线上问题和经验沉淀回需求、测试和工程规范。

## 适用路径

- 所有路径均在所属发布批次完成本节点；独立发布时单独执行。

## 路径级准入产物

| 路径 | 必需产物 |
| --- | --- |
| 快速修复 | 发布记录、监控目标、执行摘要中的风险与回滚方式 |
| 轻量需求 | 发布记录、监控目标、验证报告 |
| 复杂需求 | 发布记录、灰度结果、监控指标、回滚预案 |

## 输入

- 发布记录和当前路径对应的监控、风险与回滚信息，具体以“路径级准入产物”表为准。
- Sentry、Datadog、日志、接口指标、用户反馈和客服反馈。

## 执行

1. 观察错误率、接口失败率、性能指标、核心路径转化和用户反馈。
2. 将异常归类为需求遗漏、设计问题、实现缺陷、接口问题、环境问题或流程问题。
3. 对线上问题记录影响范围、根因、处置措施、长期修复和回归用例。
4. 将可复用结论回写到需求模板、验收清单、模块规范和组件使用规范。

## 本节点产物

- 线上监控结论。
- 复盘记录、治理任务和流程改进项。

## 完成条件

- 发布观察期结束，未发现阻断级问题或已完成处置。
- 每个线上问题都有负责人、后续动作和回归方式。

## 人工关口

模块或发布 Owner 决定问题优先级、是否扩大灰度、是否回滚和治理计划。

## 交接

将复盘结论反馈到下一轮 [需求受理](./01-requirement-intake.md)、需求澄清、验收设计和变更评估。

## 缺失处理

- 缺少发布记录或监控目标时，观察与复盘不能关闭。
- 发现阻断级问题时，按发布决策回滚或止损，并将问题作为新任务重新进入需求受理。
