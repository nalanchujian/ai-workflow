# 实施设计

将需求和影响范围转换为开发可以直接执行的最小方案；默认不修改业务代码。

## 输入

用户必须提供任务目录。根据 `03-change-assessment.json` 的路径读取准入产物：轻量需求读取需求解析与澄清；复杂需求额外读取验收清单和影响范围报告。

## 执行

1. 按页面装配、状态管理、组件、API、mapper、样式、测试划分改动。
2. 指明新增、修改和删除的文件，以及每个需求点的实现位置。
3. 定义请求参数、响应映射、状态流、错误处理、权限和刷新策略。
4. 给出验证命令、测试数据、灰度条件和回滚方案。
5. 对接口、状态码或权限的未确认项保持阻塞，不用临时猜测替代。

## 输出

写入 `/Users/j/codes/ai-workflow/tasks/<task-id>/06-implementation-plan.md`，包含文件级计划、数据流、风险、验证、回滚和待确认项。同步更新 `task.yaml`。

复杂需求完成后等待方案评审；轻量需求按任务规则记录 Owner 确认后可进入开发实现。

## 调用方式

```text
使用 $06-implementation-design

```
