# 发布管理

以可控、可观察、可回滚的方式将已批准变更发布到目标环境；默认不修改业务代码，也不直接执行生产发布。

## 输入

用户必须提供任务目录。读取当前路径对应的发布准入产物：发布批次条目、Review 报告、验证报告、发布说明和回滚方案，具体以任务路径要求为准。

## 执行

1. 确认构建、测试、审批、配置和环境依赖满足发布条件。
2. 整理发布说明、灰度范围、开关策略、监控指标和回滚指令。
3. 记录版本、时间、负责人、变更范围和发布窗口。
4. 输出灰度观察重点、异常处理方式和移交给监控复盘节点的内容。

## 输出

写入 `/Users/j/codes/ai-workflow/tasks/<task-id>/11-release-record.md`，包含发布记录、灰度结果、回滚记录或发布异常说明；同步更新 `task.yaml`。

缺少验证结果、发布范围或回滚方式时，本节点必须标记为 `BLOCKED`；AI 不直接合并生产分支或执行最终上线。

## 调用方式

```text
使用 $11-release-management
任务目录：`/Users/j/codes/ai-workflow/tasks/<task-id>`
```
