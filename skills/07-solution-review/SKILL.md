# 方案评审

在开发前确认业务、技术、依赖和风险方案，避免实现后才发现范围或接口理解错误；默认不修改业务代码。

## 输入

用户必须提供任务目录。仅复杂需求需要执行本节点；读取需求解析包、验收清单、影响范围报告、实施计划和 `path=complex` 的 `03-change-assessment.json`。

## 执行

1. 确认本期范围、排除范围、验收标准和外部依赖。
2. 审查状态流、接口契约、权限、数据兼容、公共组件和回滚方案。
3. 明确阻塞项、待确认项、负责人、验证方式和发布时间约束。
4. 输出通过、补充后通过或退回前序节点的评审结论。

## 输出

写入 `/Users/j/codes/ai-workflow/tasks/<task-id>/07-solution-review.md`，包含评审结论、批准记录、风险接受记录、待办清单和退回说明；同步更新 `task.yaml`。

缺少任一复杂需求准入产物时，本节点必须标记为 `BLOCKED`；未重新评审前不得进入开发实现。

## 调用方式

使用 $07-solution-review
