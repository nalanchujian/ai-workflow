# 当前问题对话模板

逐条确认疑点时，首次先展示节点产出，再用问题表组织当前 `RQ-*`。该文件只定义聊天格式，不保存任务进度，也不复制到 `requirement-analysis.md`。

首次进入确认时，在问题表之前输出：

| 项目 | 内容 |
| --- | --- |
| 产物 | <可点击的产物绝对路径> |
| 状态 | 草稿，节点执行中 / 已完成，等待 Gate |
| 本节点输出 | <事实、疑点、决策、线索等实际内容及数量> |
| 当前结论 | <一句话结论> |
| 风险或阻塞 | <未决项摘要；没有则写“无”> |

后续问题不重复整张产出表；Owner 确认或修改后，先说明已写入哪个 `CL-*` 及产物路径，再展示下一问题。

| 项目 | 内容 |
| --- | --- |
| 疑点编号 | `RQ-001` |
| 疑点标题 | <一句话概括需要确认的业务规则> |
| 具体问题 | <完整复述该 RQ 的问题> |
| 产生原因 | <说明资料缺口、证据冲突或无法确定的边界> |
| 推荐答案 | <本轮全部推荐 CL 的可读摘要> |
| 影响范围 | <说明该答案影响的实现或验收内容> |

表格后只附以下两种有效回复格式，并将编号替换为当前疑点编号：

- `确认 RQ-001`
- `修改 RQ-001：<最终规则>`
