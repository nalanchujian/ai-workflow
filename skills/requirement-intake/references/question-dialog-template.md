# 当前问题对话模板

逐条确认疑点时，用下表组织当前 `RQ-*` 的聊天内容。该文件只定义展示格式，不保存任务进度，也不复制到 `requirement-analysis.md`。

| 项目 | 内容 |
| --- | --- |
| 疑点编号 | `RQ-001` |
| 疑点标题 | <一句话概括需要确认的业务规则> |
| 具体问题 | <完整复述该 RQ 的问题> |
| 产生原因 | <说明资料缺口、证据冲突或无法确定的边界> |
| 推荐答案 | <本轮全部推荐 CL 的可读摘要> |
| 影响范围 | <说明该答案影响的实现或验收内容> |

表格后只附以下两种有效回复格式，并将编号替换为当前疑点编号：

- `确认 RQ-001`
- `修改 RQ-001：<最终规则>`
